Pub/sub + canonical mapping demo

One Italian-named publisher emits message.posted with fields mittente / messaggio / ora. Four subscribers receive the same event but each sees the payload in their own local vocabulary — Italian, English, compact — thanks to GlueZero’s canonical model. No pairwise adapters. No network. Just one shared canonical schema.

Library: @gluezero/gluezero@1.0.1 Topic: message.posted Canonical fields: author, body, sent_at Backend: none (pure pub/sub + mapper)

// Plugin with outputMap: canonical key ← { source: local field } broker.registerPlugin({ id: 'italian-form', canonicalSchemaId: 'message.canonical', outputMap: { author: { source: 'mittente' }, body: { source: 'messaggio' }, sent_at: { source: 'ora' }, }, }) // On Send click — top-level publish with explicit plugin source. // MapperBroker matches source.id with compiled outputMap → applies it. broker.publish('message.posted', { mittente: 'Alice', messaggio: 'Hello from GlueZero!', ora: Date.now(), }, { source: { type: 'plugin', id: 'italian-form' } })

// Italian consumer plugin: inputMap with local key → { source: canonical } broker.registerPlugin({ id: 'italian-feed', canonicalSchemaId: 'message.canonical', inputMap: { mittente: { source: 'author' }, messaggio: { source: 'body' }, ora: { source: 'sent_at' }, }, onMount: (ctx) => { ctx.broker.subscribe('message.posted', (event) => { // event.payload = { mittente, messaggio, ora } const { mittente, messaggio, ora } = event.payload appendToFeed(mittente, messaggio, ora) }) }, })
