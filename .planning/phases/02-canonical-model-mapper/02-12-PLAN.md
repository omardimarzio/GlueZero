```markdown
# @gluezero/mapper

> Canonical Model & Bidirectional Mapper per GlueZero.

[Sezioni:]
1. **Overview** — purpose, value proposition, relationship con `@gluezero/core`
2. **Install** — `pnpm add @gluezero/core @gluezero/mapper`
3. **Quick Start** — scenario meteo PRD §29 (10-15 righe Esempio funzionante)
4. **API Surface**
   - `createMapperBroker(config)` factory
   - `MapperBroker` class methods (delegati F1 + estensioni F2)
   - `registerCanonicalSchema(schema)` — esempio con `weatherSchema`
   - `registerTransform(name, fn)` — esempio `parseItalianDate`
   - `registerAlias(localField, canonicalField, opts?)` — esempio scoped vs global
5. **Mapping Resolution Order (MAP-17, chiusura PRD §39 #1)**
   ```
   1. mapping esplicito (inputMap/outputMap del plugin)
   2. alias plugin-scoped
   3. alias globali
   4. name match diretto
   5. unresolved → required:true|false policy
   ```
6. **Field Policy (VAL-08, chiusura PRD §39 #3)**
   - `required: true` + missing → `mapping.error` event
   - `required: false` + missing + `default` → applica default
   - `required: false` + missing + no default → undefined (no field nella canonical payload)
7. **Transform Failure Policy (VAL-09, chiusura PRD §39 #4)**
   - `onFailure: 'block'` (default) → mapping fallisce, `mapping.error` event
   - `onFailure: 'skip'` → field lasciato vuoto
   - `onFailure: 'fallback'` → applica `default` se definito; altrimenti come 'skip'
8. **Mapping Inspector (D-46..D-48)**
   - Estende `EventTap` di F1 (vincolo critico ARCHITECTURE.md §3.2)
   - 5 nuovi step pipeline §28: `event.source.resolved`, `event.mapped.canonical`, `event.canonical.validated`, `event.mapped.consumer`, `event.final.validated`
   - `Broker.getMappingInspector()` ritorna `{ schemas, aliases, transforms, lastErrors }`
9. **Cycle Detection (D-35)**
   - Detected al `registerPlugin` (NON a runtime publish)
   - Throw `BrokerError 'mapping.cycle.detected'` con `details: { pluginId, cycle }`
10. **Validation Adapter (D-37, D-38)**
    - Default: `valibotAdapter` (esportato da `@gluezero/mapper`)
    - Pluggable: implementa `ValidatorAdapter` interface
    - Adapter Zod/Ajv: deferred a V2
11. **Roadmap (deferred to F3-F6)**
    - F3: HTTP route gateway con server response mapping (riusa MapperEngine F2)
    - F4: realtime SSE/WS inbound normalization (riusa MapperEngine F2)
    - F5: worker payload canonicalization
    - F6: full Mapping Inspector con payload before/after per evento
12. **Constraints (PRD)**
    - Default V1: canonicalizzazione interna completa (PRD §13.5)
    - Mapping pre-compilato al `registerPlugin` (D-34) — runtime overhead minimo
    - No mapping ambiguo automatico senza configurazione esplicita (PRD §5)
13. **License** — MIT (TBD)
```
