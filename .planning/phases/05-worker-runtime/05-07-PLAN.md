## 1. Quick start

const broker = createWorkerBroker({

    topic: 'report.generation.requested',
    worker: 'report-worker',
    task: 'generateReport',
    policies: { timeout: 30_000, concurrency: 'latest-only' },

broker.registerPlugin({
  id: 'reports-plugin',

## 6. Serialization contract WK-07 (PRD §39 #11)

> **Closure ufficiale Phase 5** — questa sezione chiude esplicitamente il punto 11 della checklist PRD §39 (serializzazione messaggi worker).

### Default: structuredClone (Structured Clone Algorithm)

`@gluezero/worker` usa `postMessage` standard. Il browser applica automaticamente lo Structured Clone Algorithm (SCA) — niente `JSON.stringify`, niente `superjson` di default.

**Tipi supportati nativamente da SCA:**

| Tipo | Round-trip preservato? | Note |
|------|------------------------|------|
| `Object` (plain) | ✅ | |
| `Array` | ✅ | |
| `Date` | ✅ | `instanceof Date` preservato |
| `Map`, `Set` | ✅ | iterazione preservata |
| `ArrayBuffer`, `TypedArray`, `DataView` | ✅ | |
| `RegExp` | ✅ | flags preservati |
| `Blob`, `File`, `ImageData`, `ImageBitmap` | ✅ | |
| `MessagePort` | ✅ (transferable) | |
| `BigInt` | ✅ | |

**Tipi NON supportati (throw `DataCloneError`):**

| Tipo | Strategia raccomandata |
|------|------------------------|
| `function` | Registra via `registerTransform(name, fn)` (F2), passa `transformId: string` |
| `Symbol` | Usa string token come chiave |
| `DOM Node` (HTMLElement, Text, ...) | Estrai dato strutturato lato main |
| Class instance con prototype custom | Serializza a `{ ...obj, __type: 'ClassName' }` + reidrata lato worker |

### `assertSerializable` (D-139, D-140)

In **dev mode** (default `process.env.NODE_ENV !== 'production'`), `WorkerBridge` invoca `assertSerializable(payload)` PRE-`postMessage`. Throw sincrono `BrokerError` con shape:

\`\`\`ts
{
  code: 'worker.serialization.failed.{function|symbol|dom-node|custom-class}',
  category: 'worker',
  details: { fieldPath: 'payload.options.transform', fieldType: 'function' },
}
\`\`\`

Override:
\`\`\`ts
createWorkerBroker({ workers: { assertSerializable: 'always' | 'dev' | 'off' } })
\`\`\`

In production builds (auto-detect): disattivato → zero overhead.

### Transferable opt-in (D-141)

Per evitare il **costo della copia SCA** su payload pesanti (es. ArrayBuffer multi-MB), dichiara `transferable` come array JSONPath-like nella route:

\`\`\`ts
{
  type: 'worker',
  id: 'audio-route',
  topic: 'audio.process.requested',
  worker: 'audio-worker',
  task: 'analyze',
  transferable: ['payload.audioBuffer', 'payload.samples[*].buffer'],
}
\`\`\`

> ⚠️ **WARNING (Pitfall 7.E):** un campo `transferable` perde l'ownership lato main thread post-postMessage. `audioBuffer.byteLength === 0` immediatamente dopo `publish`. Documenta nel tuo codice consumer.

**MAI usare `JSON.stringify` lato consumer pre-publish:** F5 mantiene SCA passthrough invariato — `JSON.stringify` rompe Date/Map/Set/BigInt.
